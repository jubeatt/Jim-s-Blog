關於箭頭函式

要注意，=> xxxx 其實是有自動帶入「return」這個關鍵字的

所以如果你的寫法是像這樣 ( ) =>{ xxx }，或是這樣 function( ) { xxx }，
請務必記得要加上return

Array.prototype.filter( )

語法：array.filter(callback(element, [index], [array]), [thisArg])

作用：透過函式的運算來過濾出要留下的陣列元素

回傳值：由通過檢驗的元素所集合成的一個新的陣列

callback：該function會對陣列中每一個元素做檢驗

element：原陣列迭代處理的元素

[index]：原陣列迭代處理元素的index值

[array]：呼叫filter( )的陣列

[thisArg]：設定callback中this值的參考值（指向哪個物件）

範例

// 建立一個陣列
const words = ['Star', 'Moon', 'Dog', 'Calendar', 'Vegetable']  
// 從原陣列中篩選出字串長度 > 4 的元素
const newArr = words.filter((element) => element.length > 4)
// 輸出結果：["Calendar", "Vegetable"]
console.log(newArr)

Array.prototype.includes( )

語法：array.includes(element, [fromIndex]) (別忘了s)

作用：檢查陣列中是否包含指定的元素

回傳值：True / False

element：輸入一個值，用來查詢該值是否包含在陣列中

注意

includes( )是採用嚴格相等( === )的方式來做比較

[fromIndex]：輸入一個數值，表示要從陣列的哪一個索引值作為初始位置開始搜索（第一個的位置是0）

補充說明

如果輸入值為負數，則會從array.length + n開始搜索（等同於從倒數地n個開始向後搜索）

若輸入值 > 陣列長度，則會直接回傳False，就不會搜索陣列

若經過計算後的數值 < 0，則index會被設為0，整個陣列都會被搜索

範例

// 建立一個陣列
const animals = ['Cat', 'Dog', 'Fox', 'Hippo', 'Elephant', '1']
// 檢查陣列中是否包含字串"Cat"
const checkCat = animals.includes('Cat')
// 檢查陣列中是否包含字串"Do"
const checkDo = animals.includes('Do')
// 檢查陣列中是否包含整數 1
const checkNum = animals.includes(1)
// 輸出結果：true
console.log(checkCat)
// 輸出結果：false
console.log(checkDo)
// 輸出結果：false (整數1 != 字串1)
console.log(checkNum)

Array.prototype.indexOf( )

語法：array.indexOf(element, [fromIndex])

用途：搜尋一個元素在陣列中的索引值

回傳值：第一個被找到的元素的索引值 / 不存在則回傳「-1」

element：輸入想取得索引值的元素（不可以一次輸入多個元素）

注意

indexOf( )是採用嚴格相等( === )的方式來做比較

[fromIndex]：輸入一個數值，表示要從陣列的哪一個索引值作為初始位置開始搜索（第一個的位置是0）

補充說明

如果輸入值為負數，則會從array.length + n開始搜索（等同於從倒數地n個開始向後搜索）

若輸入值 > 陣列長度，則會直接回傳False，就不會搜索陣列

若經過計算後的數值 < 0，則index會被設為0，整個陣列都會被搜索

範例

// 建立一個陣列
const animals = ['Cat', 'Dog', 'Fox', 'Hippo', 'Elephant']
// 查詢字串"Cat"的索引值
const findIndex =  animals.indexOf('Cat')
// 查詢字串"Do"的索引值
const findDoIndex =  animals.indexOf('Do')
// 輸出結果：0
console.log(findIndex)
// 輸出結果：-1
console.log(findDoIndex)

小提醒

String物件也有一個叫做indexOf的方法，與Array的indexOf功能大致相同，但不要像我一樣不小心搞混，而搞了個大烏龍QQ

Array.prototype.forEach( )

語法：array.forEach(callback(element, [index], [array]), [thisArg])

用途：將陣列中的每一個元素傳入到callback中執行一段處理

回傳值：無

callback：對陣列中的每一個元素做一段處理

element：原陣列迭代處理的元素

[index]：原陣列迭代處理元素的index值

[array]：呼叫forEach( )的陣列

[thisArg]：設定callback中this值的參考值（指向哪個物件）

範例

// 建立一個陣列
const food = ['beef', 'fish', 'potato', 'tomato']
// 印出陣列中的每個元素及對應的索引值
food.forEach((element, index) => console.log(`${index} ${element}`))
// 輸出結果：
// "0 beef" 
// "1 fish"
// "2 potato"
// "3 tomato"

Array.prototype.find( )

語法：array.find(callback(element, [index], [array], [thisArg]))

用途：從陣列中找出符合callback函式檢測的元素值

回傳值：第一個通過函式檢測的元素值 / undefined

callback：對陣列中的每一個元素做檢測

element：原陣列迭代處理的元素

[index]：原陣列迭代處理元素的index值

[array]：呼叫forEach( )的陣列

[thisArg]：設定callback中this值的參考值（指向哪個物件）

範例

// 建立整數陣列
const numbers = [10, 20, 30, 40, 50]
// 建立字串陣列
const strings = ['10', '20', '30', '40', '50']
// 從整數陣列找出 > 整數15的元素
const findNumber = numbers.find((element) => element > 15)
// 從字串陣列找出 > 整數15的元素
const findString = strings.find((element) => element > 15)
// 從整數陣列找出 = 整數0的元素
const findZero = numbers.find((element) => element = 0)
// 輸出結果：20
console.log(findNumber)
// 輸出結果："20"
console.log(findString)
// 輸出結果：undefined
console.log(findZero)

小提醒

範例中故意多寫了一個字串陣列，並帶入函式中做檢測，這是為了要提醒各位JavaScript會做「資料轉型」，這件事情，所以在做比較時，別忘了注意資料型態的問題！

Array.prototype.findIndex( )

語法：array.find(callback(element, [index], [array]), [thisArg])

用途：find( )方法一樣，唯一的差異在於回傳值

回傳值：第一個通過函式檢測元素的索引值 / -1

callback：對陣列中的每一個元素做檢測

element：原陣列迭代處理的元素

[index]：原陣列迭代處理元素的index值

[array]：呼叫forEach( )的陣列

[thisArg]：設定callback中this值的參考值（指向哪個物件）

// 建立陣列
const numbers = [0, 10, 20, 30, 40, 50]
// 從陣列找出 > 整數15的元素
const findNumber = numbers.findIndex((element) => element > 15)
// 從陣列找出 < 整數0的元素
const findZero = numbers.findIndex((element) => element < 0)
// 輸出結果：2
console.log(findNumber)
// 輸出結果：-1
console.log(findZero)

Array.prototype.map( )

語法：array.map(callback(element, [index], [array]), [thisArg])

用途：將陣列中的每一個元素傳入到callback中執行一段處理

回傳值：經過callback運算處理後的新陣列

callback：對陣列中的每一個元素做一段處理

element：原陣列迭代處理的元素

[index]：原陣列迭代處理元素的index值

[array]：呼叫forEach( )的陣列

[thisArg]：設定callback中this值的參考值（指向哪個物件）

範例

// 建立陣列
const numbers = [2, 4, 6, 8, 10]
// 執行map方法（對陣列中每個元素 x 2）
const doubleNumbers = numbers.map((element) => element * 2)
// 印出經過運算後的新陣列
console.log(doubleNumbers)
// 輸出結果：[4, 8, 12, 16, 20]

Array.prototype.reduce( )

語法：array.reduce(callback(accumulator, currentValue, [currentIndex], [array] ), [initialValue] )

用途：對陣列中的元素做累加/減/乘/除/模，或試圖運算成一個單一值

回傳值：經過運算後得出的結果值

callback：對陣列中的每一個元素做一段處理

accumulator（累加器）：回傳callback在上一次累加所回傳的數字

累加器的初始值是？

如果有設定[initialValue]的話，累加器的初始值 = [initialValue] 設定的值
如果沒有設定[initialValue] = 直接把第一個元素拿來當作累加器（所以第一次迭代時的currentValue就會變成第二個元素）

currentValue：目前迭代處理中的元素

[currentIndex]：目前迭代處理中的元素索引值

小提醒

如果沒有設定[initialValue] 的值，會從「索引值1」的元素開始回傳（因為第一個元素會被拿去當作accumulator（累加器），反之則從「索引值0」的元素開始回傳

[array]：呼叫這個方法的array

[initialValue]：設定callback中累加器的初始值（(accumulator參數）

小提醒

它是reduce()中的參數，不是callback中的參數

如果一個空陣列使用reduce()，但沒有設定[initialValue]的值，則會發生錯誤

範例（如果陣列中包含字串資料）

// 建立整數陣列
const numbers = [2, 4, 6, 8, 10]
// 建立包含字串與整數的陣列
const mixture = ['2', 4, 6, 8, 10 ]

// 使用reduce()累加陣列元素的值
const sumOfNumbers = numbers.reduce((accumulator, currentValue) => accumulator + currentValue)
// 使用reduce()累加陣列元素的值
const sumOfMixture = mixture.reduce((accumulator, currentValue) => accumulator + currentValue)
// 設定累加器的初值
const diffSumOfNumbers = numbers.reduce((accumulator, currentValue) => accumulator + currentValue, 5)


// 輸出結果：30
console.log(sumOfNumbers)
// 輸出結果：'246810'
console.log(sumOfMixture)
// 輸出結果：35
console.log(diffSumOfNumbers)

範例（有無設定initialValue的差異）

// 建立陣列
const numbers = [1, 2, 3, 4, 5]
// 使用reduce中的callback印出每一次迭代中累加處理的過程
const result = numbers.reduce((accumulator, currentValue, currentIndex) => {
	console.log(`accumulator(${accumulator}) + currentValue(${currentValue}) = ${accumulator + currentValue}`)
	return accumulator + currentValue
})
// 註：可以看到沒有設定[initialValue]時，accumulator的值是將陣列的第一個元素來作為初始值
// 以下為每一次迭代處理的結果：
// "accumulator(1) + currentValue(2) = 3"
// "accumulator(3) + currentValue(3) = 6"
// "accumulator(6) + currentValue(4) = 10"
// "accumulator(10) + currentValue(5) = 15"

// 印出reduce的回傳值
console.log(result)
// 輸出結果：15

範例（有無設定initialValue的差異）

// 建立陣列
const numbers = [1, 2, 3, 4, 5]
// 使用reduce中的callback印出每一次迭代中累加處理的過程
const result = numbers.reduce((accumulator, currentValue, currentIndex) => {
	console.log(`accumulator(${accumulator}) + currentValue(${currentValue}) = ${accumulator + currentValue}`)
	return accumulator + currentValue
}, 5) // 設定[initialValue]的值為5

// 註：可以看到有設定[initialValue]時，accumulator的值是以[initialValue]來作為初始值
// 以下為每一次迭代處理的結果：
// "accumulator(5) + currentValue(1) = 6"
// "accumulator(6) + currentValue(2) = 8"
// "accumulator(8) + currentValue(3) = 11"
// "accumulator(11) + currentValue(4) = 15"
// "accumulator(15) + currentValue(5) = 20"

// 印出reduce的回傳值
console.log(result)
// 輸出結果：20

Array.prototype.splice( )

語法：array.splice(start, [deleteCount], [item1], [item2], [itemN]...)

用途：在陣列的指定位置刪除/新增元素

回傳值：由被刪除的元素所組成的一個新陣列

小提醒

如果你的[deleteCount]的值為0 / 負數，回傳值會得到一個「空陣列」

start：設定起始點（索引值）

[deleteCount]：要刪除的元素數量（包含start自己，並向後計算）

小提醒

如果省略 / 值超過start後面的剩餘元素數量，start後的所有元素都會被刪除

若值為0 / 負數 -> 則不會有元素被刪除

[items]：在start的前方（不是後方），加入的新元素

小提醒

如果沒有設定該參數，則只會按照start與[deleteCount]的設定去刪除元素

注意

splice( )方法會直接「對原陣列產生作用」

除非你希望的是取得的是「被刪除的元素陣列」，否則不會用到它的回傳值

範例

// 建立不完整的一周陣列
const days = ['Sun', 'Mon', 'Tue', "Fri", "Sat"]
// 在索引值3的位置插入兩個新元素
const deletedElement = days.splice(3, 0, 'Wed', 'Thur')
// 輸出結果：["Sun", "Mon", "Tue", "Wed", "Thur", "Fri", "Sat"]
console.log(days)
// []
console.log(deletedElement)

範例

// 建立文字訊息陣列
const message = ["Today", "is", "holiday"]
// 從索引值2開始刪除1個元素，並插入元素
const deletedElement = message.splice(2, 1, 'workday',)
// 輸出結果：["Today", "is", "workday"]
console.log(message)
// 輸出結果：["holiday"]
console.log(deletedElement)

Array.from( )

語法：Array.from(array-like, [callback], [thisArg])

用途：透過array-like-object（類陣列物件）或iterable-object（可迭代物件）建立一個新的Array實體

回傳值：新建立的Array實體

array-like：array-like-object（類陣列物件）或iterable-object（可迭代物件）

[callback]：迭代新陣列中的每個元素

[thisArg]：設定callback中this值的參考值（指向哪個物件）

補充說明

目前已知的iterable-object資料型別有： string、array、array-like object

註：array-like-object並非皆可迭代，要看有沒有內建@@iterator方法才算數

範例（將可迭代 / 類陣列物件轉換成陣列）

// 建立一個IIFE
(function(number1, number2) {
	// 建立可迭代物件（字串）
	const string = 'Whatever'
	// 儲存類陣列物件（function自帶的arguments物件）
	const argObj = arguments
	
	// 透過可迭代物件建立新的Array
	const createFromIterable = Array.from(string)
	// 透過類陣列物件建立新的Array
	const createFromArrayLike = Array.from(argObj)
	
	// 輸出結果：["W", "h", "a", "t", "e", "v", "e", "r"]
	console.log(createFromIterable)
	// 輸出結果：Object { 0: 100, 1: 1000 }
	console.log(argObj)
	// 輸出結果：[100, 1000]
	console.log(createFromArrayLike)

})(100, 1000) // IIFE的參數

補充說明

這裡建立了一個IIFE（Immediately Invoked Function Expression），目的是要取得function物件中的「arguments物件」。

如果你想用一般的函式宣告來定義函式也是OK的，不過此處為了省去呼叫的步驟（偷懶）所以直接以IIFE來定義。

範例（使用callback）

// 建立可迭代物件（字串）
const string = 'Whatever'
Array.from(string, function(element, index) {
  // 印出迭代元素與其索引值
  console.log(`${element} [${index}]`)
  // 輸出結果：
  // "W [0]"
  // "h [1]"
  // "a [2]"
  // "t [3]"
  // "e [4]"
  // "v [5]"
  // "e [6]"
  // "r [7]"
})
