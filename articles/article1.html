前言

作為一名前端開發者，除了摸索基礎的程式知識以外，如何提升開發效率也是件很重要的課題。
因此目前正在學習JavaScript框架的使用方式，也打算將此做為一個主題來做教學，
希望透過分享自己的學習紀錄，讓各位讀者能夠一起與我一起探索Vue.js是什麼玩意兒呢？

Vue.js是什麼？

你可能曾經聽過Vue是一個樣板框架這種說法，究竟什麼是樣板框架？讓我們一步一步來解說吧！
首先來從我們很熟悉的網頁來做解說，一個網頁的結構，簡單來說可以切割成四個項目：

  • 標記（HTML）
  • 內容（網頁中的文字、圖片、連結等...資料）
  • 樣式（CSS）
  • 程式（JavaScript）

Vue的作用就是負責把網頁中「內容」的部分給抽起來，換句話說，這些內容就代表了資料。
這樣的用意是什麼呢？這是為了讓管理資料與畫面的方式能夠更簡單。

還是摸不著頭緒？其實你可以仔細想想，這個問題就類似於「我們如何用CSS來控制網頁的樣式佈局」是一樣的道理。
舉例來說，假使今天有10個網頁要做樣式佈局，我相信有遠見的你會知道，「啊！我們可以應該要把相同樣式的部分寫成一份CSS來統一管理，免得之後要維護時搞死自己！」。

是的，就是這樣簡單的想法，Vue.js出現了。
以往我們想要控制網頁的布局時，我們第一個想到的是CSS。
那麼如果我們想要控制網頁的內容的時候呢？就交給Vue吧！

還是好抽象啊？能舉個例子嗎！

就拿這個部落格來舉例吧，假設我想修改左上角的個人頭像，那麼我該怎麼做呢？
「簡單啊，把HTML中頭像的src的路徑更新不就好了？」
是的，在一般的情況下，這樣是最快又簡單的作法，但問題是，如果今天有10個頁面都要做更新呢？
沒有錯，我們就得去更新每一個HTML中的內容。
雖然這是一種解決方式，但顯然並不是一個聰明的作法。

那麼，什麼才是聰明的做法？想想我們前面說的，「用CSS來管理網頁的樣式佈局」的邏輯，
既然我們可以把網頁的樣式佈局抽出來用CSS統一管理，那同樣地，我們也把網頁的內容抽出來統一管理就好了呀！
這也就是Vue.js想達成的事情，雖然目前市面上並沒有設計出一個專門用來處理HTML資料的的工具，但幸運的是，我們有JavaScript。

相信各位都知道，JavaScript可以用來控制網頁中的內容，所以我們只要準備好想要呈現的資料，再用JavaScript去把資料給填入到HTML中，就可以做到我們所說的「統一管理」這件事。不過要是用原生的JavaScript來做這些事情，那光用想的也是一件大工程吧！

不用擔心，現在Vue.js已經幫你做好這件事情了，我們只要照著Vue.js的操作方式來使用，就能夠做輕鬆做到統一管理資料這件事。不過還是要提醒各位，不論是什麼框架，它的本質並不會改變，就好比Vue.js，說穿了，它也還是用JavaScript寫出來的東西。

馬上來Vue一個頁面吧！

Step1. 準備樣板（HTML頁面）

還記得我們提過Vue.js是一個樣板框架嗎？
所以在我們使用它之前，我們要先準備好一個樣板，也就是HTML頁面，如下： 

<div id="app">
	<h1></h1>
	<img>
	<a></a>
</div>

補充解說

你可能發現我們並沒有在HTML中寫入任何的資料、甚至是屬性。
這是因為資料的部分，待會兒我們會使用Vue來做處理，所以這裡的HTML就如我們所說的，它只是個「樣板」而已，

Step2. 載入Vue.js

載入Vue的方式有很多種，這裡提供給一個最簡單的方式，使用CDN來引入，你只需要在你的HTML中加入下面的程式碼即可：

<script src="https://unpkg.com/vue@3.0.11"></script>
// @後面代表的是Vue.js的版本，可以自行指定。

（這只是其中一種方式，想了解更多可以參考Vue.js的官方說明

Step3. 建立Vue的物件實體

建立好Vue之後，接著就能開始來撰寫需要的程式碼囉！

小提醒

你可以建立一個新的.js外部檔案，或者是直接使用<script>標籤來撰寫程式碼

讓我們先看看下面的程式碼：

const vm = Vue.createApp({
	data() {
		return{
			title: '標題',
			srcPath: '路徑',
			linkPath: '連結'
		}
	}
});
vm.mount('#app')

要使用Vue的第一件事情是要先建立出Vue的物件實體，所以你看到上面使用Vue.creatApp({...})來建立一個Vue的物件實體，接著把它存入vm裡面。
而({...})中的內容就是我們想要設定的東西，data()的部分就是我們要提供給Vue.js儲存的資料，
最後的mount('#app')，是代表要與Vue做連結的DOM（HTML元件）。

補充說明

const是在ES6規範下的一種新的變數宣告方式。他的特點是宣告時就必須指定一個值，且這個值之後不能在被改寫。

data( ) { ... } 是在ES6規範下的一種新的物件方法撰寫方式。這只是一種更簡潔的寫法，但與以往的data : function( ) { .... }的意義是相同的。

Step4. 把資料寫入到HTML中

在Step2的時候，我們已經利用mount('#app')來把Vue與HTML元件做了連結，所以接著就可以使用Vue的標記方式，來把我們希望呈現的資料寫入到HTML中，如下： 

<div id="app">
	<h1>{{ title }}</h1>
	<img :src="src">
	<a :href="link"></a>
</div>

{{title}}

是指把Vue中的title資料，寫到這個位置的意思。這裡使用的其實是雙花括{{ }}的縮寫方式，實際上他原本的標記是長這個樣子： 

<h1 v-text="title"></h1>

v-text是透過屬性值的方式來從Vue中取得對應的資料。
不論你想用v-text或是{{ }}，兩者得到的結果都是相同的。

注意

v-text的用途主要用來輸出純文字。如果現在想輸出的內容包含了HTML標籤，請改用Vue提供的另外一種方式：v-html來操作。如果你嘗試使用v-text來輸出HTML標籤，那標籤的部分將會自動被轉譯成「字符實體」。
什麼是字符實體？就是把HTML標籤的部分轉譯成可以正常顯示的字符，這是為了防止被瀏覽器解析成HTML標籤。
有點複雜？沒關係，總之你只要記得他就是會變成一串可以顯示的純文字就行了！

:src="src" :href="href"

事實上這也是一種縮寫的方式，是以「:」來表示，原本的標記方式如下： 

<img v-bind:src="src">

v-bind:src我們可以這樣子來解讀：

  • 要綁在哪一個HTML屬性？
  • 要輸入的資料是？（Vue裏頭的什麼資料）

所以上面的意思就等同於告訴Vue ➝ 綁在<img>的src上，並且在裡面輸出Vue中的src資料
